DP LC1320 Minimum Distance to Type a Word Using Two Fingers

Posted on 2020-02-19

Answer Analysis: https://leetcode.com/problems/minimum-distance-to-type-a-word-using-two-fingers/discuss/477659/4%2B-DP-Solutions

Top-down DP

  • The process of solving this problem is a binary tree
    • Root node: word
    • Left branch: the finger one types the current character
    • Right branch: the finger two types the current character
  • how to decide which finger =
    the sum of distance if using that finger to type the current character
    =
    the distance between the current finger and the current character +
    the sum of distance of typing next character based on the situation that using the current finger typing the current character
  • So we can see there is a recursion.

Bottom-up DP

  • Use the given longest length of word
    • For each character, go through all combinations of two fingers and calculate the sum of distance between each finger and the current character.
    • dp[pos][i][j] = min(dp[pos + 1][to][i] + cost(j, to), dp[pos + 1][to][j] + cost(i, to));
      • To decide the current finger, we need to compare the two possible results:
        • finger 1 at position i moves to position to, so finger 2 is still at position j
        • finger 2 at position j moves to position to, so finger 1 is still at position i
      • So we can know the design of this DP data structure is for each position pos, the previous character could be any letter i, then another finger will be at any letter j.

Optimization of above two types of DP

  • Itâ€™s unnecessary to distinguish finger one and finger two.
  • For the current character, there are only two choices:
    • using the same finger as the previous character
    • using another one

Algorithm in Data Mining - Sampling algorithm

Posted on 2019-09-04

Pseudo code

public int[] samplingAlgorithm(int[] allNum, int sampleSize) {
int[] sample = new int[sampleSize];
for (int i = 0;i < sampleSize; i++) {
sample[i] = allNum[i];
}

int t = sampleSize + 1;
Random random = new Random();
while (t < allNum.length) {
    int rnm = random.nextInt(t);
    if (rnm < sampleSize) {
        sample[rnm] = allNum[t];
    }
    t++;
}

return sample;

}

LC 843 Guess the word

Posted on 2019-05-27

The algorithm is:

  1. Choose a guessed word from wordlist randomly
  2. Check the number m of matched letters by calling master.guess
  3. Refresh wordlist.
    (1) get the number n of matched letters between the current word in the original wordlist and the guessed word.
    (2) if n equals to m, then add the current word into the new wordlist

Why does this algorithm work?

Think about that. If the first guessed word has 0 matched letters. That means all letters in the guessed word are not in the secret word. Then the new wordlist has all words which have 0 matched letters against the guessed word. So we can see that the accuracy of getting the answer is approved by getting the next guessed word in the new wordlist.

Binary Search - LC 410 Split Array Largest Sum

Posted on 2019-05-24

Analysis

Find the largest sum of all elements of a subarray among all subarrays.
So the answer is between the maximum element of this array(min_result) and the sum of all elements(max_result). And thatâ€™s based on the condition that elements are non-negative.

  • if # of subarrays is # of elements, then the largest sum among of these elements should be the largest element of this array.
  • if # of subarrays is 1, then there is only one answer.

So, the problem is converted to a new problem that we need to find a proper number between min_result and max_result.

Then another question is that if we try to use binary search to solve this new problem, how could we make sure that the array can be cut into m subarrays.

Using binary search to find a proper maximum sum for every subarray and count the number of subarrays.
If the current number of subarrays is smaller than or equal to the given number m, then this maximum sum needs to be smaller.
If the current number of subarrays is larger than m, then this maximum sum needs to be larger.
